# Fleet Perception Demo

A playable version of a passed predictive-maintenance validation gate: 30 NASA
C-MAPSS turbofan engines the system **never saw in training**, ageing in
calendar time. Each engine's health index climbs from green → amber → red as it
degrades, and the detail panel shows *which sense* is surprised — the
modality-decomposed triage the product sells.

The backing data is generated by the exact validation pipeline (same parser,
same healthy-only training, same seed/split), so this demo **is** the passed
gate (FD001 holdout: ρ=0.762, AUC=0.996), visualized — not a re-run with
different knobs.

Open `index.html` in a browser — no server needed. Press **Play** to watch the
fleet age; click any tile for its per-sense breakdown, health-index trajectory,
and plain-language narration.

## Honesty notes

- Trained on **healthy operation only** (first 30% of life, 70 *other*
  engines), **zero failure labels** — the turbofan was onboarded as just
  another device type via a generated sense-mapping.
- Surprise is baselined so **1.0 = average healthy**. Status bands: amber ≥3×,
  red ≥6×. The displayed index is a causal 5-cycle trailing mean.
- **The amber/red bands are display calibration, not a validated alarm
  policy** — the gate's headline metrics (ρ, AUC) are threshold-free; a
  production alarm policy would be set per fleet on held-out healthy data.
- 28 of 30 engines start nominal and are flagged before failure (median 59
  cycles ahead). **2 read hot from cycle 1** — genuine unit-to-unit variation,
  shown honestly rather than hidden; in production you'd burn-in per asset.
